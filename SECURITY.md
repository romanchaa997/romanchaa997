# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in any of our projects, please report it to us responsibly. We appreciate your efforts to responsibly disclose your findings.

### How to Report

1. **Do not open public GitHub issues** for security vulnerabilities
2. Email security details to: security@romanchaa997.dev
3. Include proof-of-concept or detailed reproduction steps
4. Allow 48 hours for initial response
5. Provide a reasonable timeline for a fix (typically 30-90 days)

### What to Expect

- Acknowledgement of your report within 48 hours
- Regular updates on our progress
- Credit in security advisory (if desired)
- Coordinated disclosure timeline

## Security Practices

### For Developers

- Always sign commits with GPG keys
- Use strong, unique passwords
- Enable two-factor authentication on GitHub
- Review code for common vulnerabilities (OWASP Top 10)
- Keep dependencies updated and audit regularly

### For Users

- Enable 2FA on accounts using our services
- Use strong, unique passwords
- Report suspicious activity immediately
- Keep software updated
- Review API key permissions regularly

## Supported Versions

Security updates are provided for:
- Current major version (all minor versions)
- Previous major version (latest patch only)

## Known Vulnerabilities

None currently disclosed. Check GitHub Security Advisory database for our project.

## Dependencies

We actively monitor dependencies for vulnerabilities using:
- GitHub Dependabot
- npm audit
- pip safety
- SNYK scanning
